## 1. 저장 전략 결정

### 1.1 스토리지 옵션 비교

| 기준 | localStorage | IndexedDB | 서버 DB |
|------|-------------|-----------|---------|
| 용량 | ~5MB | ~50MB+ | 무제한 |
| 구조화 데이터 | 불가 (JSON 직렬화) | 가능 (객체 저장) | 가능 |
| 쿼리/필터 | 불가 | 가능 | 가능 |
| 로그인 필요 | 불필요 | 불필요 | 필요 |
| 구현 복잡도 | 낮음 | 중간 | 높음 |
| 브라우저 간 동기화 | 불가 | 불가 | 가능 |
| SSR 호환 | 주의 필요 | 주의 필요 | 가능 |

### 1.2 MVP 결정: localStorage 채택

**이유:**
- 로그인 없이 즉시 이력 유지 가능
- BlogPost 타입 구조가 단순 (JSON 직렬화 충분)
- 구현 속도가 빠르고 디펜던시 없음
- content(markdown) 필드가 길 수 있으나, 최대 개수 제한으로 관리 가능

**trade-off 인정:**
- 브라우저/기기 간 동기화 불가 — 로그인 기능 도입 시 서버 DB로 마이그레이션
- 5MB 제한 → 최대 저장 개수로 보완

### 1.3 최대 저장 개수 제한

- **기본 최대: 20개**
- 저장 시 현재 개수가 20개이면 가장 오래된 항목 1개 자동 삭제 후 저장 (FIFO)
- 사용자에게 "오래된 이력이 자동 삭제됩니다" 토스트 알림 제공
- 계산 근거: BlogPost 1개 평균 ~10KB (본문 ~8KB + 메타데이터) → 20개 = ~200KB (여유 충분)

### 1.4 브라우저 간 동기화

- MVP 범위 밖 — 동일 브라우저/기기 내에서만 유지
- 향후 로드맵: 사용자 계정 + 서버 DB 마이그레이션 시 지원

## 2. 이력 목록 화면 기능 정의

### 2.1 표시할 정보 (카드 단위)

| 요소 | 설명 | 우선순위 |
|------|------|---------|
| 제목 (title) | 최대 2줄, 넘치면 말줄임 | 필수 |
| 주제 (topic) | 배지(badge) 형태로 표시 | 필수 |
| 생성일 (createdAt) | "3일 전", "2026-03-01" 형식 | 필수 |
| 글자수 | content 길이 기반 "약 1,200자" | 선택 |
| 썸네일 | images[0] URL (없으면 기본 이미지) | 선택 |
| 참고자료 수 | references.length "참고 3건" | 선택 |

### 2.2 정렬 옵션

- **기본: 최신순** (createdAt DESC)
- 선택: 제목순 (title ASC 알파벳/가나다)
- UI: 드롭다운 셀렉트 (`정렬: 최신순 ▼`)

### 2.3 검색/필터 기능

- **MVP 포함: 텍스트 검색** — 제목 + 주제 대상 (debounce 300ms)
- MVP 제외: 날짜 범위 필터, 태그 필터 (추후 추가)
- 검색어 입력 시 실시간 결과 업데이트 (클라이언트 사이드 필터링)

### 2.4 삭제 기능

- **단건 삭제**: 카드 우측 상단 `×` 버튼 → 확인 없이 즉시 삭제 + 실행취소(undo) 토스트 5초
- **전체 삭제**: 목록 상단 "전체 삭제" 버튼 → 확인 다이얼로그 필수 ("정말 삭제하시겠습니까? 복구 불가")
- 삭제 후 목록 즉시 업데이트

### 2.5 빈 상태(Empty State) 처리

```
[일러스트 또는 아이콘]
아직 생성된 블로그 글이 없습니다.
첫 번째 글을 생성해 보세요!
[→ 글 생성하기] 버튼
```

## 3. UX 플로우

### 3.1 이력 목록 → 상세 진입 흐름

```
이력 목록(/history)
  └─ 카드 클릭
      └─ /result/[id] 로 라우팅
          └─ localStorage에서 id로 BlogPost 로드
              ├─ 성공: 결과 화면 렌더링 (읽기 전용)
              └─ 실패: "이력을 찾을 수 없습니다" + 목록으로 돌아가기 버튼
```

- 결과 페이지(/result/[id])는 이력에서 진입 시 "← 이력으로 돌아가기" 버튼 표시
- 새로 생성된 결과는 "← 새 글 작성하기" 버튼 표시 (진입점 구분)

### 3.2 메뉴에서 이력 접근 방법

**채택: 헤더 탭 방식**

```
[AI 블로그 생성기]    [새 글 작성]  [이력]
```

- 헤더 내비게이션에 "이력" 탭 추가
- 모바일: 햄버거 메뉴 내 "이력" 항목
- 활성 탭에 하이라이트 표시 (현재 페이지 인디케이터)

**선택 안 한 이유:**
- 사이드바: 콘텐츠 영역 줄어들고 글쓰기 앱에 부적합
- 드롭다운: 이력 기능의 중요도를 고려할 때 숨겨두기엔 중요함

### 3.3 결과 페이지에서 저장 시점

**채택: 자동 저장**

- /result/[id] 페이지 로드 시 즉시 localStorage에 자동 저장
- 저장 완료 시 우측 하단 토스트: "이력에 저장되었습니다 ✓"
- 이미 저장된 id면 업데이트하지 않음 (중복 저장 방지)
- **별도 저장 버튼 없음** — 인지 부담 제거

**자동 저장 채택 이유:**
- 사용자가 저장을 잊을 경우 이력 소실 방지
- 생성형 AI 결과물은 재생성 시 동일 결과 보장 불가 → 즉시 보존 필요

## 4. 사용자 스토리

### Story 1
**[반복 사용자]로서 [이전에 생성한 글을 다시 읽기 위해] [이력 목록]을 원한다.**

수용 기준:
- /history 페이지에 생성한 블로그 글 목록이 카드 형태로 표시된다
- 각 카드에 제목, 주제, 생성일이 표시된다
- 카드를 클릭하면 /result/[id]로 이동하여 전체 내용을 볼 수 있다
- localStorage가 비어있으면 빈 상태 메시지와 생성 링크가 표시된다

---

### Story 2
**[콘텐츠 기획자]로서 [특정 주제의 글을 찾기 위해] [이력 검색 기능]을 원한다.**

수용 기준:
- 이력 목록 상단에 텍스트 검색 입력창이 있다
- 입력 후 300ms 이내에 제목과 주제를 대상으로 필터링 결과가 표시된다
- 검색 결과가 없으면 "검색 결과가 없습니다" 메시지가 표시된다
- 검색창 초기화 시 전체 목록으로 돌아간다

---

### Story 3
**[사용자]로서 [불필요한 이력을 정리하기 위해] [개별 삭제 및 전체 삭제]를 원한다.**

수용 기준:
- 각 카드에 삭제 버튼(×)이 있고 클릭 시 즉시 목록에서 제거된다
- 삭제 후 5초간 실행 취소 토스트가 표시되며, 클릭 시 복원된다
- "전체 삭제" 버튼 클릭 시 확인 다이얼로그가 표시된다
- 확인 후 모든 이력이 삭제되고 빈 상태 화면이 표시된다

---

### Story 4
**[사용자]로서 [생성한 글이 자동으로 보관되도록] [자동 저장 기능]을 원한다.**

수용 기준:
- /result/[id] 페이지 진입 시 해당 BlogPost가 localStorage에 자동 저장된다
- 저장 완료 시 "이력에 저장되었습니다" 토스트 메시지가 표시된다
- 동일 id의 글은 중복 저장되지 않는다
- 이력이 20개를 초과하면 가장 오래된 1개가 자동 삭제되며 토스트로 알린다

---

### Story 5
**[사용자]로서 [최근 글과 오래된 글을 구분하여 탐색하기 위해] [정렬 기능]을 원한다.**

수용 기준:
- 이력 목록 상단에 정렬 드롭다운(최신순 / 제목순)이 있다
- 기본 정렬은 최신순(최근 생성일 우선)이다
- 정렬 변경 시 목록이 즉시 재정렬된다
- 선택된 정렬 옵션이 드롭다운에 표시된다

## 부록: 데이터 스키마 (localStorage)

```typescript
// localStorage key: 'blog-history'
// value: JSON.stringify(BlogPost[])

interface BlogPost {
  id: string;           // nanoid or uuid
  topic: string;        // 사용자 입력 주제
  title: string;        // AI 생성 제목
  content: string;      // markdown 본문
  images: ImageItem[];  // [{ url, alt, credit }]
  references: Reference[]; // [{ title, url, snippet }]
  createdAt: string;    // ISO 8601
}

// 저장 순서: 배열 맨 앞 = 최신 (index 0 = newest)
// 삭제 기준: 배열 맨 뒤 (index 19 = oldest)
```
